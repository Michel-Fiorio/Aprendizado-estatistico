Tarefa 2. Utilizando o mesmo conjunto de dados BreastCancer realize análises de classificação. Para tanto, a variável ‘Class’ (última variável do conjunto) é nossa variável dependente. Utilize validação cruzada para avaliar os modelos ajustados. Faça um relatório. Tal relatório deve ser entregue de maneira organizada discutindo os resultados encontrados.

Realizaremos análise do banco de dados Breast Cancer utilizando as técnicas de Análise Discriminante Linear e Análise Discriminante Quadrática. Poderemos verificar qual técnica apresentará modelos com melhores resultados de classificação.

Análise Discriminante Linear:

library(MASS)

discrilinear.bc = lda(Class ~ ., data = bc[,2:11], prior = c(1,1)/2 , CV = TRUE)

# Obter a classificação resultante do modelo para o conjunto de dados

Classif.linear = discrilinear.bc$class
# Gerar a matriz de confusão do modelo
confusao.linear = table(bc$Class, Classif.linear)
confusao.linear
##            Classif.linear
##             benign malignant
##   benign       436         8
##   malignant     18       221
# Calculando taxa de erro aparente do modelo

tea.linear = 1 - (sum(diag(confusao.linear))/sum(confusao.linear))
tea.linear
## [1] 0.03806735

Análise discriminante Quadrática:

discriquad.bc = qda(Class ~ ., data = bc[,2:11], CV = TRUE)

# Obter a classificação resultante do modelo para o conjunto de dados

Classif.quad = discriquad.bc$class
# Gerar a matriz de confusão do modelo

confusao.quad = table(bc$Class, Classif.quad)
confusao.quad
##            Classif.quad
##             benign malignant
##   benign       416        28
##   malignant      6       233
# Calculando taxa de erro aparente do modelo

tea.quad = 1 - (sum(diag(confusao.quad))/sum(confusao.quad))
tea.quad
## [1] 0.04978038

Análise dos resultados: realizamos a análise discriminante dos dados no formato linear e quadrático. Utilizamos na geração dos modelos a validação cruzada do tipo ‘leave-one-out’. Foi gerada a matriz de confusão e calculado a taxa de erro aparente dos modelos. Pudemos observar que a taxa de erro foi muito pequena, de apenas 3,8% no modelo da técnica linear e de 4,97% no modelo da técnica quadrática. Portanto o modelo discriminante linear foi mais eficiente na classificação dos dados.

Tarefa 3. Utilize o conjunto Boston Housing Dataset para predizer o valor médio das casas ocupadas pelo proprietário (medv) por meio de técnicas de predição. Faça um relatório. Tal relatório deve ser entregue de maneira organizada discutindo os resultados encontrados.

Primeiramente obtemos o arquivo de dados BreastCancer, avaliamos o conteúdo e limpamos dados faltantes (NA), se necessário. Também particionamos os dados em dois conjuntos, o de treinamento e o de teste. Cada um com 50% das amostras.

# Obtendo e analisando as características do banco de dados
dados.boston = Boston

#Particionar o conjunto de dados (Treinamento e validacao)
set.seed(1)
treinamento.boston = sample (1:nrow(dados.boston), nrow(dados.boston)/2)

Modelo utilizando árvore de regressão / random forest com os dados de treinamento:

library(randomForest)
## randomForest 4.7-1.1
## Type rfNews() to see new features/changes/bug fixes.
## 
## Attaching package: 'randomForest'
## The following object is masked from 'package:ggplot2':
##

arvore.boston = randomForest(medv ~ .,data=dados.boston , subset=treinamento.boston ,
                      mtry=4, importance =TRUE)

Medida de qualidade do modelo acima com os dados de teste. Cálculo da raiz do erro quadrático médio:

predicao.boston = predict(arvore.boston,newdata=dados.boston[-treinamento.boston,])
teste.boston=dados.boston[-treinamento.boston ,"medv"]
REQM_arvore.boston <- sqrt(mean((predicao.boston-teste.boston)^2))
REQM_arvore.boston
## [1] 4.320301

Modelo utilizando técnica de regressão linear múltipla com os dados de treinamento:

regressão.boston = lm(medv ~ ., data = dados.boston, subset = treinamento.boston)

Medida de qualidade do modelo acima com os dados de teste. Cálculo da raiz do erro quadrático médio:

predicao.regressão.boston = predict(regressão.boston,newdata=dados.boston[-treinamento.boston,])
REQM_regressão.boston <- sqrt(mean((predicao.regressão.boston-teste.boston)^2))
REQM_regressão.boston
## [1] 5.182783

Análise dos resultados: Montamos dois modelos diferentes de regressão dos dados Boston Housing utilizando as técnicas de regressão linear múltipla e ramdom forest. OS modelos foram treinados utilizando os dados de treinamento. Para validação do modelo utilizamos os dados de teste e calculamos a eficiência do modelo através do cálculo da raiz do erro quadrático médio(REQM). Os resultados obtidos indicam que o modelo criado pelo random forest ficou ligeiramente melhor ajustado que o modelo de regressão linear.
